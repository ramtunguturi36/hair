========================================
1) PROJECT OVERVIEW
========================================
Project Name (working): AI Hair Analysis Platform

One-line summary:
A full-stack AI-powered hair care platform where users can sign in, buy credits, upload/capture hair photos for analysis, get personalized routines and product recommendations, track progress, chat with an AI consultant, and receive notifications.

Core value:
It is not just an image classification demo. It is an integrated product flow with personalization, monetization, and admin controls.

========================================
2) TECH STACK (CODEBASE-VERIFIED)
========================================
Frontend:
- React 18 + TypeScript + Vite
- React Router (nested dashboard routes)
- Tailwind CSS + custom utility/theme styling
- Clerk authentication
- React Three Fiber / Drei / Three.js for 3D hero visualization
- Stripe client SDK for checkout redirection
- Teachable Machine model integration for image classification path
- Fetch + Axios (both currently used)

Backend:
- Node.js + Express (ESM)
- Multer (memory storage) for file upload
- Google Generative AI SDK (Gemini)
- Stripe server SDK
- dotenv, cors
- fs/path JSON-file persistence for MVP storage

Deployment/runtime posture:
- Backend has Dockerfile (Node 18-alpine)
- Backend can serve frontend dist if build exists
- Env-driven keys and URLs

========================================
3) IMPLEMENTATION ARCHITECTURE
========================================
High-level architecture:
- Frontend handles UI, auth state, navigation, and user interactions.
- Backend provides all API endpoints for analysis, data persistence, payments, notifications, and admin summaries.
- Data storage is currently file-based JSON (history, profiles, progress, notifications, payments, flags, saved products).

Major frontend modules:
- Public pages: Home, About, Contact, Pricing, Login/Signup
- Dashboard pages: Analysis, Photo, Pricing, History, Consultation, Hair Loss, Style Guide, Profile, Saved, Progress, Notifications, Admin
- Shared systems: CreditContext, API utility wrapper, UI components

Major backend API categories:
- Payments: checkout session + payment logging
- User data: history, profile, saved products, progress, notifications
- Admin: dashboard totals + recent logs + open flags
- AI: analyze hair image, chat consultant, analyze face shape, analyze risk

========================================
5) WORKFLOW (HOW APP WORKS)
========================================
Typical user flow:
1. User authenticates (Clerk).
2. User buys a credits pack from pricing page (Stripe checkout session).
3. On success redirect, app adds credits and logs payment.
4. User uploads or captures photo.
5. App deducts credits and calls AI analysis endpoint.
6. User sees result + routine + recommendations.
7. Analysis gets stored in history.
8. User can track progress via before/after entries.
9. User can ask follow-up questions via AI consultant chat.

========================================
6) WHAT IS SPECIAL / HOW IT DIFFERS
========================================
Key differentiation points for review:
1. End-to-end productization:
   Most projects stop at image prediction. This project includes full user lifecycle (auth, wallet, payment, retention pages, admin).

2. Multi-AI journey (not single endpoint):
   AI is used across 4 distinct value points:
   - hair image analysis
   - risk assessment
   - face-shape style guidance
   - conversational consultant

3. Personalization beyond classification:
   User profile attributes (concerns, goals, allergies, budget) influence routine generation and recommendations.

4. Reliability thinking:
   Gemini model fallback strategy is implemented for high-demand/service-unavailable scenarios.

5. Governance/quality control:
   AI response flagging + admin visibility is included.

6. Monetization readiness:
   Credit economy + Stripe integration + payment logging + low-credit notifications.

Suggested one-liner to say in panel:
"The novelty is not one isolated AI model; it is the integration of AI + personalization + monetization + moderation into a usable product lifecycle platform."

========================================
9) LIMITATIONS / RISKS (IMPORTANT TO MENTION HONESTLY)
========================================
- JSON file storage is not scalable for concurrency and integrity
- Several routes trust userId from client instead of strict server auth checks
- Credit update flow is partly client-driven and can be hardened
- Mix of fetch + axios increases inconsistency
- Hardcoded localhost redirects in payment flow reduce deploy portability
- Minimal formal automated test coverage currently

========================================
10) FUTURE ROADMAP
========================================
Priority improvements:
1. Move storage to PostgreSQL or MongoDB
2. Implement strict server-side authorization checks on all user-scoped routes
3. Add Stripe webhook as source of truth for payment finalization
4. Move all credit mutation logic behind server-validated operations
5. Standardize API client usage patterns (choose one: fetch wrapper or axios)
6. Add API + integration tests for critical flows
7. Add robust markdown sanitization strategy where HTML rendering is used
8. Improve recommendation ranking using feedback loop and outcomes over time
